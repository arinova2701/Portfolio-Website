# Arinova — Portfolio Website

> In a world of AI-generated slop, I designed, built, and deployed this myself.

Live: http://bit.ly/43J9F4n

---

## What this is

A personal portfolio for Maneesh Ari (Arinova) — ML Engineer, AI & Automation specialist, backend developer, and designer based in Hyderabad, India.

No templates. No website builders. No drag-and-drop. Just HTML, CSS, and JavaScript written line by line, with a lot of DevTools, a lot of `Cmd+Z`, and a healthy amount of stubbornness.

---

## Stack

Vanilla HTML, CSS, JavaScript. No frameworks, no build tools, no npm dependencies except EmailJS for the contact form.

---

## What's in it

- **Matrix cursor** — katakana + English character trail that follows your mouse, occasionally glitching red
- **ARINOVA glitch effect** — RGB channel-split drop shadows on the display type, fires every 12 seconds
- **ASCII art textures** — personal photos converted to ASCII and clipped inside the letterforms
- **Split-flap role animation** — cycles through identities (ML Engineer, Designer, Builder, Storyteller...)
- **Liquid signature** — SVG `feTurbulence` + `feDisplacementMap` filter giving the Maneesh Ari script a gooey, flowing quality
- **Lily animations** — wind-sway with irregular keyframe stops and a shimmer filter pulse, two lilies never in sync
- **Typing animation** — terminal status line types itself out on load
- **Contact form** — wired to EmailJS, actually sends to my inbox
- **Matrix video background** — subtle ambient layer with screen blend mode
- **Mobile responsive** — separate layout logic for phone viewports

---

## Why no framework

Because the point was to understand what I was building. Every CSS property, every animation keyframe, every JS event listener was written and debugged manually. Using React or Tailwind would have been faster. It also would have taught me less.
